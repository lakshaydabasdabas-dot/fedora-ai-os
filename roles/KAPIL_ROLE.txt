Issue 6: Build MCP Gateway health dashboard
  Description: HTML page at GET /dashboard. Table showing each backend
  with name, URL, status (healthy/degraded/down). Status from health
  checks. Circuit breaker state. Recent request count and error rate.
  Auto-refresh every 5 seconds via meta tag. No JavaScript required.

    - Request count and error rate shown (last 60s)

Issue 7: Implement retry with exponential backoff
  Description: Retry failed requests up to 3 times with delays of 1s, 2s,
  4s. Only retry retryable errors (502, timeout, connection error). Do
  NOT retry non-retryable errors (400, 401, 403, 404).
  Difficulty: Medium
  Acceptance Criteria:
    - retry_call(func, max_attempts=3) retries on failure
    - Backoff: 1s, 2s, 4s
    - Non-retryable errors (400, 401, 403, 404) not retried
    - Each retry attempt logged with attempt number
    - After max attempts, returns original error
    - Test: mock fails twice, succeeds third time

Issue 8: Implement circuit breaker state machine
  Description: Three-state circuit breaker. CLOSED: requests pass through.
  3 consecutive failures → OPEN. OPEN: all requests fail fast for 30s
  without calling backend. After 30s → HALF_OPEN: one probe request.
  Success → CLOSED. Failure → OPEN (reset timer).
  Difficulty: Medium
  Acceptance Criteria:
    - CircuitBreaker class with states: CLOSED, OPEN, HALF_OPEN
    - 3 consecutive failures → OPEN
    - OPEN for 30 seconds → HALF_OPEN
    - HALF_OPEN + success → CLOSED
    - HALF_OPEN + failure → OPEN (timer reset)
    - Thread-safe (or asyncio-safe)
    - Test: all state transitions

Issue 9: Implement dead-letter queue to audit memory
  Description: When retry is exhausted or circuit breaker is OPEN, write
  the failed request to audit memory (via Jatin's audit_store module).
  Include: original request, backend name, error, timestamp, retry count.
  Difficulty: Medium
  Acceptance Criteria:
    - Failed request written to audit memory
    - Fields: request_body, backend_name, error_message, timestamp,
      retry_count, circuit_breaker_state
    - Format matches audit memory schema
    - Test: failed request appears in audit query results

Issue 10: Write API documentation
